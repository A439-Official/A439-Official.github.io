CSS

核心概念

盒模型解析

box-sizing: 控制元素尺寸计算方式（content-box/border-box）

标准盒模型：width = 内容宽度（不包含padding/border）

IE盒模型：width = 内容宽度 + padding + border（通过box-sizing: border-box实现）

选择器体系

类选择器(.class)：最常用选择器

ID选择器(#id)：唯一标识选择器（慎用！）

属性选择器([attr])：精准定位元素

伪类选择器(:hover)：状态选择器（支持动态交互）

伪元素选择器(::before)：内容生成技术

布局系统

传统布局

float: 创建浮动布局（需配合clear使用）

position: 绝对定位（absolute/relative/fixed/sticky）

display: 传统显示模式（block/inline/inline-block）

现代布局

Flexbox布局：

  • display: flex 激活弹性容器
  • flex-direction: 主轴方向
  • justify-content: 主轴对齐
  • align-items: 交叉轴对齐
  • flex-wrap: 换行控制

Grid布局：

  • display: grid 创建网格容器
  • grid-template-columns: 定义列轨道
  • grid-template-rows: 定义行轨道
  • grid-gap: 网格间距
  • grid-area: 区域命名

视觉呈现

高级属性

clip-path: 自定义形状裁剪

transform: 2D/3D变形（translate/scale/rotate/skew）

filter: 滤镜效果（blur/grayscale/contrast等）

backdrop-filter: 背景模糊效果（毛玻璃效果）

@media: 媒体查询（响应式设计核心）

动画系统

@keyframes: 定义关键帧动画

animation: 复合属性（控制动画名称/时长/延迟等）

transition: 过渡效果（配合:hover等状态使用）

现代特性

变量系统

--custom-property: 定义CSS变量

var()函数: 使用CSS变量

预处理支持

Sass/Less: 嵌套规则、混合宏、函数等特性

PostCSS: 未来CSS语法转换工具

性能优化

  1. 避免使用@import引入样式表
  2. 合理使用will-change属性
  3. 限制复杂选择器的使用
  4. 采用CSS containment隔离样式
  5. 使用CSSOM优化动态样式

最佳实践建议

  1. 优先使用CSS变量维护主题系统
  2. 复杂布局优先采用Grid布局方案
  3. 动画优先使用transform/opacity实现
  4. 移动端适配采用rem+vw单位方案
  5. 避免!important过度使用（调试时除外）
  6. 重要样式采用内联方式提升关键渲染路径
