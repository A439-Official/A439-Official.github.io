GitHubPage

教程：从零搭建个人静态站点

GitHub Pages 是 GitHub 提供的免费静态网站托管服务，支持自定义域名、HTTPS 加密和持续集成部署，特别适合个人博客、项目文档和作品集展示

第一步：账号准备

1.1 注册 GitHub 账号

访问 GitHub 注册页面，建议使用企业邮箱注册（需验证）

1.2 启用双重验证

进入 Settings > Account security 启用 2FA，提升账户安全性

第二步：创建站点仓库

2.1 仓库命名规范

必须命名为 username.github.io（严格区分大小写），其中 username 为你的 GitHub 用户名

2.2 初始化仓库

推荐初始化时添加 README.md 文件，可勾选 Add a README file 选项

第三步：内容开发

3.1 基础文件结构

username.github.io/
├── index.html       # 站点入口文件
├── css/
│   └── style.css    # 样式表
├── js/
│   └── main.js      # 脚本文件
└── images/
    └── logo.png     # 图片资源

3.3 使用 Jekyll 构建（高级）

  1. 在仓库根目录创建 _config.yml 配置文件
  2. 支持 Markdown 文档自动转换
  3. 内置主题选择：进入 Settings > Pages > Theme Chooser

第四步：部署配置

4.1 基础部署

提交代码后，访问 https://username.github.io 即可查看站点（部署可能需要1-2分钟）

4.2 自定义域名

  1. 在域名服务商处配置 CNAME 记录指向 username.github.io
  2. 在仓库根目录创建 CNAME 文件（无扩展名），内容为你的域名

4.3 HTTPS 强制

进入 Settings > Pages 启用 Enforce HTTPS 选项

第五步：高级优化

5.1 SEO 优化

  • 添加 <meta name="description"> 标签
  • 生成 sitemap.xml 并提交至 Google Search Console

5.2 性能优化

  • 启用 Gzip 压缩（通过 .github/workflows/pages.yml 配置）
  • 使用 <picture> 标签实现响应式图片

5.3 持续集成

通过 GitHub Actions 实现自动化部署：

name: Deploy GitHub Pages
on:
  push:
    branches:
      - main
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public

常见问题

Q: 修改后页面未更新？

A: 清除浏览器缓存或尝试强制刷新（Ctrl+F5），部署延迟最长可达10分钟

Q: 如何使用自定义404页面？

A: 在仓库根目录创建 404.html 文件，GitHub Pages 会自动识别
